# [Epic] 요구사항 - 동물들을 관리하기가 힘들어요

## 에픽 설명
- 현재는 동물 정보를 한눈에 보거나, 등록하거나, 상태를 바꾸는 흐름이 정리되어 있지 않아 관리가 어렵다.
- 사용자가 동물을 쉽게 등록하고, 목록으로 보고, 상태를 확인하고, 상호작용할 수 있는 흐름이 필요하다.

---

## Story 1. 새로운 동물을 등록할 수 있다
### 설명
- 사용자는 관리 대상 동물을 시스템에 등록할 수 있어야 한다.

### 완료 기준 (Acceptance Criteria)
- [x] 사용자가 동물 이름을 입력할 수 있다.
- [x] 사용자가 동물 나이를 입력할 수 있다.
- [x] 사용자가 동물 종류(강아지/고양이)를 선택할 수 있다.
- [x] 등록이 완료되면 목록에 반영된다.

---

## Story 2. 등록된 동물 목록을 볼 수 있다
### 설명
- 사용자는 현재 관리 중인 동물들을 한눈에 확인할 수 있어야 한다.

### 완료 기준 (Acceptance Criteria)
- [ ] 등록된 동물들이 번호와 함께 출력된다.
- [ ] 각 동물의 이름, 종류, 나이가 표시된다.
- [ ] 등록된 동물이 없으면 안내 문구가 출력된다.

---

## Story 3. 동물에게 먹이를 줄 수 있다
### 설명
- 사용자는 동물의 배고픔 상태를 관리하기 위해 먹이를 줄 수 있어야 한다.

### 완료 기준 (Acceptance Criteria)
- [x] 사용자가 목록에서 동물을 선택할 수 있다.
- [x] 선택한 동물에게 먹이를 줄 수 있다.
- [x] 먹이를 주면 배고픔 수치가 감소한다.
- [x] 배고픔 수치는 최소값 아래로 내려가지 않는다.

---

## Story 4. 동물과 놀아줄 수 있다
### 설명
- 사용자는 동물의 기분 상태를 관리하기 위해 함께 놀 수 있어야 한다.

### 완료 기준 (Acceptance Criteria)
- [x] 사용자가 목록에서 동물을 선택할 수 있다.
- [x] 선택한 동물과 놀아줄 수 있다.
- [x] 놀아주면 행복도가 증가한다.
- [x] 행복도는 최대값을 넘지 않는다.

---

## Story 5. 동물의 상태와 특성을 확인할 수 있다
### 설명
- 사용자는 동물의 현재 상태를 보고, 종류에 따른 반응도 확인할 수 있어야 한다.

### 완료 기준 (Acceptance Criteria)
- [x] 사용자가 특정 동물을 선택할 수 있다.
- [x] 이름, 종류, 나이, 배고픔 수치, 행복도를 확인할 수 있다.
- [x] 동물의 울음소리를 들을 수 있다.
- [x] 강아지와 고양이는 서로 다른 울음소리를 출력한다.

# [Epic] 다양한 동물로 확장해서 운영하기 (Story 5개 버전)

## 에픽 설명
- 강아지/고양이 중심 구조를 조류/포유류/파충류까지 확장한다.
- 동물별 능력/먹이/사육사/통계를 “한 기능씩” 단계적으로 붙여 운영 가능하게 만든다.

---

## Story 1. 중간 분류(Bird/Mammal/Reptile)를 도입해 확장 가능한 상속 구조로 바꿀 수 있다
### 설명
- 다양한 동물을 깔끔하게 추가하기 위해 `Animal` 아래에 중간 분류 계층을 만든다.

### 완료 기준 (Acceptance Criteria)
- [x] `Animal`은 추상 클래스로 유지된다.
- [x] `Bird`, `Mammal`, `Reptile` 추상 클래스가 추가된다.
- [x] 기존 동물(`Dog`, `Cat`)은 `Mammal`을 상속하도록 변경된다.
- [x] 기존 기능(등록/목록/먹이/놀기/상태/울음)이 깨지지 않는다.

---

## Story 2. 추가 동물 6종을 등록하고, 각자 다른 울음소리를 낼 수 있다
### 설명
- 요구된 동물들을 실제로 추가하고, 종류별 특성이 드러나도록 울음소리를 구현한다.

### 완료 기준 (Acceptance Criteria)
- [x] 조류: `Eagle`, `Penguin` 등록 가능
- [x] 포유류: `Lion`, `Elephant` 등록 가능
- [x] 파충류: `Snake`, `Turtle` 등록 가능
- [x] 등록 후 목록에 이름/종류/나이가 정상 반영된다.
- [x] 모든 동물의 울음소리가 서로 다르게 출력된다.

---

## Story 3. 특별 능력을 인터페이스(Flyable/Swimmable)로 분리해 실행할 수 있다
### 설명
- 동물별 고유 능력을 상속이 아니라 인터페이스로 분리하여 조합 가능하게 만든다.

### 완료 기준 (Acceptance Criteria)
- [x] `Flyable(fly())`, `Swimmable(swim())` 인터페이스가 추가된다.
- [x] `Eagle`은 `Flyable`을 구현한다.
- [x] `Penguin`은 `Swimmable`을 구현한다.
- [x] 능력 실행 메뉴에서 선택한 동물의 능력을 실행할 수 있다.
- [x] 능력이 없는 동물을 선택하면 안내 메시지가 출력된다.


## Story 4. 먹이 시스템(Food)과 동물별 선호 먹이를 도입해 적합 먹이 여부를 판정할 수 있다
### 설명
- 먹이 종류를 모델링하고, 동물마다 “먹어도 되는 것/선호하는 것” 규칙을 적용한다.

### 완료 기준 (Acceptance Criteria)
- [x] `Food` enum(또는 클래스)이 추가된다. (예: MEAT, FISH, PLANT, NUT)
- [x] 각 동물은 선호 먹이(또는 허용 먹이 목록)를 가진다.
- [x] 사용자는 먹이를 선택해서 동물에게 줄 수 있다.
- [x] 적절한 먹이면 배고픔이 감소한다.
- [x] 부적절한 먹이면 경고 메시지가 출력되고 배고픔 감소가 제한/무효 처리된다.

---

## Story 5. 사육사(ZooKeeper)와 통계(요약)를 도입해 운영 상태를 한눈에 확인할 수 있다
### 설명
- “누가 관리하는가(사육사)”와 “현재 운영 상태(통계)”를 붙여 관리력을 올린다.

### 완료 기준 (Acceptance Criteria)
- [ ] `ZooKeeper` 클래스가 추가되고 전문 분야를 가진다. (조류/대형동물/파충류 등)
- [ ] 전문 분야가 맞으면 먹이주기/놀아주기 효과가 증가한다.
- [ ] 동물 종류별 개체 수를 출력할 수 있다.
- [ ] 전체 동물 평균 행복도를 계산해 출력할 수 있다.
- [ ] 배고픈 동물 목록(기준치 이하)을 출력할 수 있다.
- [ ] 동물원 현황 요약(총 수/분류별 수/평균 행복도/배고픈 수)을 한 번에 출력할 수 있다.
