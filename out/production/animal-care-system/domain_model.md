# 도메인 모델 요약
---
## Animal
모든 동물의 공통 행동을 담는다.

### 속성
- `name` : 동물 이름
- `age` : 동물 나이
- `hunger` : 배고픔 수치
- `happiness` : 행복도

### 행위
- 자신의 종류를 반환한다: `getType()`
- 목록에 표시할 기본 정보를 제공한다: `summary()`
- 먹이를 받아 배고픔 수치를 감소시킨다: `feed()`
- 함께 놀아 행복도를 증가시킨다: `play()`
- 상태를 편한 문자열로 만든다: `status()`
- 울음소리를 낸다: `makeSound()`
-  분류(Bird/Mammal/Reptile)는 중간 분류 클래스가 확장.

### 규칙
- 이름은 비어 있으면 안 된다.
- 나이는 0 이상이어야 한다.
- 배고픔 수치는 0 이상이어야 한다.
- 먹이를 주면 배고픔 수치는 감소해야 한다.
- 놀아주면 행복도는 증가해야 한다.
- 행복도는 최대값(100)을 넘으면 안 된다.
- 울음소리는 종류별로 달라야 한다.

### 검증
- `name == null` 이면 안 됨
- `name.isBlank()` 이면 안 됨
- `age < 0` 이면 안 됨
- 초기 배고픔 수치는 0 이상이어야 함
- 초기 행복도는 0 이상 100 이하여야 함

## (추가) Bird / Mammal / Reptile
`Animal` 아래의 중간 분류(추상 클래스)

### 역할
- 큰 분류를 고정한다. (조류/포유류/파충류)
- 구체 동물은 분류를 직접 관리하지 않고, 상속으로 자동 결정된다.

### 규칙
- `Bird`의 분류는 항상 "조류"
- `Mammal`의 분류는 항상 "포유류"
- `Reptile`의 분류는 항상 "파충류"

## Dog, Cat
`Animal`의 구현체

### 속성
- 부모인 `Animal`의 속성을 그대로 사용

### 행위
- `getType()`을 각자 다르게 구현
    - `Dog` → "강아지"
    - `Cat` → "고양이"
- `makeSound()`를 각자 다르게 구현
  - `Dog` → "멍멍!"
  - `Cat` → "야옹!"

### 규칙
- 타입/울음소리는 구현체가 고정적으로 책임진다.

### 검증
- 생성 시 `Animal`의 공통 검증을 그대로 따름

## (추가) 능력 인터페이스 (Flyable / Swimmable 등)
동물의 “고유 능력”을 상속이 아니라 **조합**으로 붙이기 위한 규칙

### 대상
- `Flyable` : `fly()`를 수행할 수 있는 동물
- `Swimmable` : `swim()`을 수행할 수 있는 동물

### 규칙
- 능력은 "분류"가 아니라 "행동"이므로 인터페이스로 분리한다.
- 구체 동물은 필요한 능력만 구현한다. (예: Eagle=Flyable, Penguin=Swimmable)
- 능력을 구현하지 않은 동물은 능력 실행 메뉴에서 안내 메시지가 출력된다.

### 검증/판정(실행 시점)
- 선택된 동물이 해당 인터페이스를 구현했는지 확인 후 실행한다.
  - 구현 O → 능력 실행
  - 구현 X → "특별 능력이 없습니다" 안내

## Zoo
동물들을 관리하는 관리자 역할.

### 속성
- `animals : ArrayList<Animal>`

### 행위
- 동물을 등록한다
- 등록된 동물 목록을 보여준다
- 목록이 비었는지 확인한다: `isEmpty()`
- 선택한 동물에게 먹이를 준다: `feedAnimal(int index)`
- 선택한 동물과 놀아준다: `playWithAnimal(int index)`
- 선택한 동물의 상태를 보여준다: `showAnimalStatus(int index)`
- 선택한 동물의 울음소리를 들려준다: `hearAnimalSound(int index)`

### 규칙
- 등록된 동물은 목록에 저장되어야 한다.
- 목록 출력 시 **번호가 1부터** 시작해야 한다.
- 목록 출력 시 각 항목은 **이름/종류/나이**가 표시되어야 한다.
- 목록이 비어 있으면 안내 문구가 출력되어야 한다.
- 잘못된 번호로 동물을 선택하면 먹이를 줄 수 없다.
- 잘못된 번호로 동물을 선택하면 놀아줄 수 없다.

### 검증
- `null` 동물은 등록하면 안 됨
- 선택 인덱스가 범위를 벗어나면 안 됨

# 책임 분리

---

## Main
### 책임
- 사용자 입력 받기
- 메뉴 출력
- 입력값을 기반으로 `Dog` 또는 `Cat` 생성
- `Zoo`에 등록 요청
- `Zoo`의 목록 출력 기능 호출
- 먹이를 줄 동물을 선택받아 `Zoo`에 요청
- 놀아줄 동물을 선택받아 `Zoo`에 요청

 결론: **입력/출력(UI) 책임**

---

## Animal, Dog, Cat
### 책임
- 동물이라는 개념과 상태를 가짐
- 배고픔 수치를 스스로 안전하게 관리함
- 먹이를 받을 때 자신의 상태를 변경함
- 놀아줄 때 자신의 상태를 변경함

 결론: **도메인 Animal 책임**

---

## Zoo
### 책임
- 여러 동물을 보관하고 관리
- 목록 출력 규칙(번호/표시 형식/빈 목록 처리)을 책임짐
- 선택된 동물을 찾아 먹이주는/놀아주기 흐름을 연결함

결론: **목록 관리 책임**
